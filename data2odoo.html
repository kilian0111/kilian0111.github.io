DATA2ODOO

Contexte

Créée par les adhérents du groupement en 1967, GEDEX est une coopérative de commerçants et le 1er groupement de distributeurs français de matériaux et d’équipements pour la construction, la rénovation, l’aménagement et la décoration.

GEDEX anime deux enseignes complémentaires :
o GEDIMAT, premier groupement de négociants indépendants en matériaux de construction et bricolage
o GEDIBOIS, enseigne dédiée au bois-panneaux créée fin 2008

De plus GEDIMAT a voulu uniformiser les systèmes informatiques du réseau des adhérents et a donc fait appel à INTERWAY.
Aujourd’hui, certains adhérents bénéficient d’un SI se résumant à l’utilisation de fichiers Excel.

Objectif et Problématique

Pour ce projet il faut donc réaliser la reprise du fichier client pour l’entreprise adhérente « NEGOMAT » située dans la région lyonnaise.
De plus il existait déjà une solution qui permettait de lire les données du fichier Excel et qui effectuait une requête pour rajouter ses données dans une base de données PostgreSQL.

Donc les besoins et objectifs étaient de corriger les éventuelles erreurs mais aussi de faire évoluer cette solution pour contrôler les données enregistrées. Pour cela nous avions une liste de demandes.

Les technologies

  • Odoo l'ERP de GEDIMAT ou le PGI en français (Progiciel de Gestion Intégré) se définit comme un groupe de modules relié à une base de données unique.
  • c#
  • Github
  • PostgreSQL
  • PGAdmin 4 : nous avons utilisé PGAdmin pour avoir une interface graphique de notre base de données PostgreSQL

Gestion de projet

Pour ce projet on a travaillé en groupe et donc utilisé github pour la gestion de projet.

Présentation de la solution

En résumé nous avons un fichier CSV qui regroupe tous les clients de NEGOMAT que nous devons intégrer à ODOO

Ainsi le premier champ sert à renseigner le chemin où se trouve le fichier CSV à intégrer à la base PostgreSQL.

Le deuxième et le troisième champ servent à renseigner l'adresse où se trouve la base de données et le nom de celle-ci.

Le quatrième champ est optionnel est nous permet d'envoyer un rapport par mail sur les clients insérés, ainsi que d'éventuelles anomalies.

De plus pour ce projet il nous a aussi fallu réaliser des tests unitaires pour être sûr que les clients intégrés ont tous les informations nécessaires.

Bilan

Cette application m’a permis de mieux comprendre la nécessité de bien séparer les différentes actions dans des class ; ainsi, quand une erreur survient, on sait plus facilement où se situe l’erreur.
De plus, grâce à cette application, j’ai appris à travailler avec des tickets et en groupe avec GitHub ce que j’ai dû appliquer durant mon stage.
