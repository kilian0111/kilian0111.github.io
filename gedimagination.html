Gedima'Gination

Contexte

Ce projet se passe dans le même contexte que DATA2ODOO. Nous nous retrouvons ainsi chez Interway, une ESN (Entrepise de Services Numériques).
Ici, Interway travaille pour le groupe GEDIMAT. La société NEGOMAT vient tout juste d'entrer dans le réseau d'adhérents de GEDIMAT. Pour fêter cet événement, NEGOMAT a décidé d'organiser un jeu concours nommé GEDIMA'GINATION.

Principe du jeu

Le jeu est ouvert aux clients de NEGOMAT.
Pour participer, il suffit de s’inscrire et poster une photo d’une réalisation faite à partir de matériaux achetés chez NEGOMAT.

Dans un deuxième temps, les clients de passage dans le magasin voteront pour leurs trois réalisations préférées.
Les photos des réalisations seront consultables sur un écran TV.
Les clients pourront donc utiliser une tablette installée dans le magasin pour saisir leurs votes.
A la fin du concours, seront désignés les trois gagnants qui seront récompensés par des bons d’achat.

Objectif et Problématique

L'objectif était d'avoir un site internet et une application mobiles qui répondent aux critères ci-dessous :

APPLICATION WEB :

Cette application permettra :

- au gérant du magasin, de publier le règlement et le calendrier du jeu concours ainsi que les résultats (classement des trois meilleures réalisations),
- aux participants, de s’inscrire et de poster leur photo accompagnée d’un titre, d’un descriptif et d’une date de réalisation des travaux.

Les données de l’application web seront hébergées dans une base de données MySQL. dédiée.


- APPLICATION MOBILE :

Avant le début des votes, les données caractérisant les photos des participants seront importées dans une base de données embarquée dans l’application (base SQLite) via un web service.

Pendant toute la période de vote, les votes des clients seront enregistrés dans la base de données embarquée.

Lorsque la période de vote sera terminée, toutes les données relatives aux votes seront envoyées dans la base MySQL de l’application web via un web service.

Pendant toute la période des votes, l’application mobile fonctionnera en mode déconnecté.

Gestion de projet

Pour réaliser les différentes applications nous avons utilisé la conduite de projet en mode agile.
Nous avions donc des users stories à réaliser en sprint d'une semaine.
De plus, pour travailler en groupe nous avons utilisé Github ainsi qu'un serveur de base de données distant sur un Raspberry pi.
